Secondly, weâre not specifying a dockerfile, which means the default Dockerfile for each container will be used instead of eg Dockerfile.dev

Thirdly, the nginx container is going to be listening on port 80, not 3100, so we can access it via url. Fourthly, we have a restart always command. As weâre running these in the background, if we reboot or there is a problem with anything we want the containers to restart automatically.
